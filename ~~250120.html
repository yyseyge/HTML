CSS 스타일 시트 개요

CSS 적용 타겟 샘플문장입니다.

Selector는 CSS스타일 시트의 이름 규칙. 스타일 시트의 블록은 중괄호로 처리한다. CSS스타일은 프로퍼티와 값을 쌍으로 설정한다.CSS는 200개 정도 프로퍼티(속성)이 있음 계속 추가되는 중

CSS주석처리

CSS의 주석은 /* ~~~~*/ 로 주석가능


CSS스타일 시트를 적용하는 방법 3가지

스타일 적용 방법1: head내에 style태그로 구성하는 방법

스타일 적용 방법2: 태그의 style속성에 직접 적용하는 방법

태그에 직접적용하는. 이게 2번방법

스타일 적용 방법3: 외부 스타일 시트를 import 혹은 link로 불러와서 사용하는 방법

스타일 태그를 통해 스타일 시트 만들때는 head태그 내에 style태그를 작성한다.

CSS3스타일 시트만 떼어내어 CSS확장자를 가진 파일을 만들어 스타일 시트를 넣어 저장하고 필요할때 불러와서 사용할 수 있음.

link태그를 이용한 외부 스타일 시트 적용

link의 href속성에 적용하려는 외부 스타일 시트 파일의 경로, 파일명을 작성한다.

type속성에 text/css속성값을 적용한다.

rel속성에 stylesheet속성 값 적용한다.

link태그는 종료태그가 없고 head태그 내에 작성한다.

CSS의 규칙

CSS스타일 시트에는 규칙이 있다.

1.CSS스타일은 부모 태그로부터 상속된다. 자신을 둘러싸는 태그를 부모태그 혹은 부모 요소(element)라고 하는데, 부모 태그에 적용된 스타일은자식 요소에 상속 적용 됨.

스타일 합치기와 오버라이딩

브라우저는 각 태그에 대한 디폴트 스타일을 가지고 있음. 따라서 개발자가 따로 지정하는 스타일이 없으면 브라우저 기본 디폴트 값을 통한 스타일이 적용 됨. 예)pedding,margin는 0으로 초기화 직적 해줌

특정 태그에는 4가지 종류의 스타일 시트가 적용 될 수 있음

1.브라우저 디폴트 2.외부 CSS파일 시트 3.내부 스타일 시트 4. 인라인 스타일 시트

스타일 합치기란 4가지 스타일 시트가 태그에 동시 적용될때 스타일이 합쳐져 적용됨을 의미

스타일 오버라이딩이란 덮어쓰기의 의미로 4가지 루트 중 같은 속성(예를 들어 폰트크기)가 여러번 적용되면 우선순위가 높은 스타일시트로 덮어씌워짐

우선순위 1번부터 4번으로 갈수록 높아짐

CSS의 단위

CSS 스타일 시트에서 단위를 작성하는 (크기, 두께, 높이 등) 단위에는 px, %, em 등 여러 방법 있음

em:배수 브라우저가 폰트 사이즈 16px => 1em 폰트 사이즈의 1배

rem:배수 브라우저가 폰트 사이즈 16px => 1em 폰트 사이즈의 1배

rem과 em의 차이 : rem과 em은 모두 상대적 크기 단위, 요소의 크기를 부모 요소의 크기에 비례하여 조정한다. rem은 루트(html)의 폰트 크기를 기준으로 한다. em은 부모 요소의 폰트 크기를 기준으로 한다. 루트요소(html)의 폰트 크기가 16px(기본값)일때 1rem은 16px만큼을 의미하고 em은 부모로부터 상속받은 폰트 크기에 따라 배율이 설정되는 것

CSS를 이용한 태그의 배치

HTML 요소들은 웹 페이지에 작성한 순서대로 출력되는 것이 기본 값

기본 구조에 css를 적용하여 배치를 변경할 수 있음

배치관련 속성 display/left/top/bottom/right/float/z-index/visibility/overflow 등 있음

예를 들어 div태그의 기본 속성이 블럭형태인것을 display속성을 활용해서 span태그처럼 인라인 형태로 변경 할 수 있음

대표적인 블록 태그 : p/div/ul

대표적인 인라인 태그 : span/a/img

앞에서 개발자 도구를 통해 봤던 박스 모델에 따라 브라우저는 각 HTML태그를 하나의 박스로 다룬다

기본적으로 브라우저는 블록 태그는 블록박스로, 인라인 태그는 인라인 박스로 구성한다.

블록은 새로운 라인에서 브라우저의 좌측 끝부터 우측 끝까지 영역을 차지하려는 속성

인라인은 해당 인라인 태그가 포함하는 컨텐츠 만큼의 영역을 차지하는 속성

DIV내부태그

기존 블럭 속성과 인라인 속성을 혼용한 형태인 inline-block이라는 형태도 display속성을 통해 설정가능

display에 적용할 수 있는 속성값 3가지 비교

1.display:block 속성 설정=> 항상 새라인에서 시작한다. 블록 박스 내에만 배치. 앞에 다른 요소 배치 불가능, width와 height로 크기 조절가능하다. padding/margin/border 조절가능

2.display:inline 속성 설정 => 새 라인에서 시작하지 않음. 모든 박스 내 배치 가능, 옆에 다른 요소 배치 가능, width와 height로 크기 조절 불가능, margin-top/ margin-bottom 조절 불가능

3.display: inline-block 속성 설정 => 새 라인에서 시작하지 않음, 모든 박스 내 배치 가능, 옆에 다른 요소 배치 가능, width와 height 조절 가능, padding/margin/border설정 가능

normal flow

기본 HTML 선언 순서 구조대로 출력되는 형태를 노멀 플로우라고 함

position 속성을 사용하면 normal flow를 무시하고 원하는 위치에 박스를 배치할 수 있음

position속성의 값: static(기본값: normal flow)//relative/absolute/fixed

static 정적배치상태에서는 left/top/right/bottom등 값을 설정할 수 없음

relative 상대배치: static 기본 배치에서 left/top/bottom/right 프로퍼티 값을 준 만큼 상대적으로 이동한 위치에 배치하는 방법

absolute 절대 배치: HTML 태그를 절대 위치에 배치 가능. 절대위치는 l/r/t/b/ 으로 값을 정하고 부모 태그 안에서의 상대 좌표이다(공중에 뜬다)

absolute 포지션을 적용하면 div가 일반적인 문서 흐름에 따른 위치를 차지하지않고 공중에 떠서 배치 된다. 즉 absolute로 설정된 요소는 문서 레이아웃 흐름에서 제외된다. 그저 부모의 위치 기반의 상대 위치로 배치됨.

fixed 고정 배치: 스크롤을 내리거나 브라우저 크기를 변경해도 항상 같은 위치에 고정되는 배치 방법, 고정배치는 절대 배치와 달리 뷰포트에 고정 시켜 브라우저에 항상 보이게 한다.(스크롤을 따라다닌다)
